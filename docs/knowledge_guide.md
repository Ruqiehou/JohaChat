# Joha 知识库管理文档

## 1. 知识库概述

Joha 的知识库系统是一个本地 RAG（检索增强生成）引擎，基于 **BM25 算法**进行文档检索。知识库内容会在 LLM 生成回复时自动被检索并注入到上下文中，让机器人具备领域知识回答能力。

### 1.1 核心特性

| 特性 | 说明 |
|------|------|
| 存储格式 | 分片式 JSON，每片 100 条文档 |
| 检索算法 | BM25（Okapi BM25 变体） |
| 中文分词 | jieba |
| 热重载 | 支持运行时添加/删除，无需重启 |
| 自动扩展 | 分片自动创建，无限扩容 |

### 1.2 存储位置

```
joha/storage/txt/
├── knowledge_0001.json    # 第 1 片（文档 1-100）
├── knowledge_0002.json    # 第 2 片（文档 101-200）
├── knowledge_0003.json    # 第 3 片（文档 201-300）
└── ...
```

## 3. 知识格式规范

### 3.1 单条知识结构

```json
{
  "id": "uuid",
  "question": "问题文本",
  "answer": "答案文本",
  "created_at": "2026-05-23T16:00:00"
}
```

### 3.2 知识编写建议

**好的知识示例：**

```
/知识库添加 群号是什么？|本群群号是 123456789
/知识库添加 如何安装 Joha？|参考文档：https://github.com/xxx/joha#快速开始，通过 pip install -r requirements.txt 安装依赖
/知识库添加 机器人叫什么？|机器人叫约哈（Joha），可以叫我小助手
```

**不推荐的写法：**

```
/知识库添加 a|b              # 太简短，检索效果差
/知识库添加 非常长的文章...   | 过于冗长，建议拆分
```

### 3.3 批量添加

如果需要批量添加知识，可以手动编辑 JSON 分片文件：

```json
[
  {
    "id": "自定义-id-1",
    "question": "问题1",
    "answer": "答案1"
  },
  {
    "id": "自定义-id-2",
    "question": "问题2",
    "answer": "答案2"
  }
]
```

保存到新分片文件后，发送 `/知识库刷新` 热加载。

## 4. 程序化操作

### 4.1 使用 Python API

```python
from joha.tools.knowledge.core import KnowledgeBase

kb = KnowledgeBase()

# 搜索
results = kb.search("什么是 Joha", top_k=5)
for r in results:
    print(f"问题: {r['question']}")
    print(f"答案: {r['answer']}")
    print(f"分数: {r['score']}")

# 添加
kb.add_document(
    question="机器人版本",
    answer="当前版本：v1.0.0"
)

# 删除
kb.delete_document("文档id")

# 保存到磁盘
kb.save()

# 统计
stats = kb.get_stats()
print(f"总文档数: {stats['total_documents']}")
print(f"分片数: {stats['shard_count']}")
```

### 4.2 批量导入脚本

```python
"""批量导入知识到知识库"""
import json
from joha.tools.knowledge.core import KnowledgeBase

kb = KnowledgeBase()

# 从 JSON 文件导入
with open("knowledge_data.json", "r", encoding="utf-8") as f:
    data = json.load(f)

for item in data:
    kb.add_document(
        question=item["question"],
        answer=item["answer"]
    )

kb.save()
print(f"已导入 {len(data)} 条知识")
```

## 5. 检索效果优化

### 5.1 提高检索准确率

1. **关键词匹配**：问题和答案都包含核心关键词
2. **问题多样化**：同一个知识可以用不同方式提问
3. **分词优化**：使用标准中文表达，避免缩写

### 5.2 添加同义问题

如果需要覆盖同一知识的多种问法，可以添加多条知识：

```
/知识库添加 机器人怎么关闭？|发送 /全局关闭 切换到被动模式
/知识库添加 怎么让机器人不说话？|发送 /全局关闭 切换到被动模式
/知识库添加 如何静默机器人？|发送 /全局关闭 切换到被动模式
```

### 5.3 检索参数调整

BM25 检索参数通常无需修改，但如需调整可在 `knowledge/core.py` 中配置：

```python
# BM25 参数
k1 = 1.5  # 词频饱和度参数
b = 0.75  # 文档长度归一化参数
```

## 7. 检索流程说明

当群内出现新消息时，知识库检索的流程：

```
用户消息: "Joha 怎么安装？"
        ↓
    jieba 分词: ["Joha", "怎么", "安装"]
        ↓
    BM25 检索 → 匹配 knowledge_*.json
        ↓
    返回 Top-3 相关文档
        ↓
    注入到 LLM 上下文:
    [系统消息] 相关参考知识：
    1. 问题：如何安装 Joha？答案：...
    2. 问题：Joha 环境要求？答案：...
        ↓
    LLM 生成回复（参考知识库内容）
```

## 8. 常见问题

**Q: 搜索为什么找不到我添加的知识？**

- 确保已执行 `/知识库刷新`
- 检查关键词分词结果是否匹配
- 尝试用不同的关键词搜索

**Q: 知识库数据会丢失吗？**

- 数据存储在 `joha/storage/txt/` 目录中
- 建议定期备份此目录
- 升级时不会覆盖此目录

**Q: 最大能存储多少知识？**

- 理论上无上限，每片 100 条自动创建新分片
- 建议控制在 10000 条以内以保证检索速度

**Q: 如何删除单条知识？**

- 目前需要手动编辑 JSON 文件删除，然后 `/知识库刷新`
- 或通过代码 API: `kb.delete_document(doc_id)`
